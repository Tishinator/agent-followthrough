2. Watchdog Decision Logic

  1. Load active tasks.
  2. For each task, inspect current state from the best available source:
    • exec/process status
    • OpenClaw session/subagent status
    • known completion markers
  3. If completed or failed, send a final update and mark resolved.
  4. If still running and overdue for check-in, send a progress update and refresh lastCheckinAt.
  5. If observation is inconclusive, send a degraded-status update only if overdue.

3. Notification Rules

  • Completion: immediate
  • Failure/blocker: immediate
  • Running/overdue: every configured interval, default 2–3 minutes
  • No duplicate updates if status is unchanged and interval has not elapsed

4. Proof Strategy by Design

  • Registry file proves tracked tasks exist.
  • Scheduler logs prove checks actually executed.
  • Message/update logs prove follow-up notifications were sent.
  • State transitions prove completion/failure handling.
