Revision 3 Change Log

Revision 3 is a targeted closure pass that addresses the remaining implementation-gate issues identified in the Rev 2 review.

Remaining IssueRevision 3 Response
Emitter gate not closedArchitecture now names the Phase 1 emitter explicitly: openclaw message send --channel telegram --target <id> --message "...".
Scheduler choice still ambiguousRevision 3 explicitly selects OpenClaw native cron as the Phase 1 scheduler.
Registration mechanism not fully specifiedRevision 3 defines agent-follow-up as a local Node.js CLI in the project directory with an invocation path and command contract.
Conditional observability undefinedRevision 3 removes vague conditional language from Phase 1 and narrows support to process-backed and marker-backed tasks only.
Unknown threshold N not definedRevision 3 sets the default unknown-state threshold to 2 watchdog cycles.
resolution_status vs last_observed_status unclearRevision 3 adds an explicit semantic distinction between observed transient state and resolved terminal outcome.
Raw PID as observable_id is unsafeRevision 3 changes the process-backed observable identity to an owned marker/sentinel path, with PID as optional secondary evidence only.
